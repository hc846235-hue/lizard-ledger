# 记账系统故障排查指南

## 📱 当前问题汇总

### 问题 1: 登录后还是不能上传到云端

**可能原因:**
1. 数据库权限设置不正确
2. CloudBase 环境配置问题
3. 网络连接问题

**解决方案:**

#### 1.1 检查数据库权限

在 CloudBase 控制台中:
1. 进入【数据库】→【bills 集合】→【权限设置】
2. 确认权限规则为:
   ```
   {
     "read": "auth != null",
     "write": "auth != null"
   }
   ```
3. 点击【保存】

#### 1.2 检查用户登录状态

在浏览器开发者工具 Console 中查看日志:
- 查找 `========== 开始从云端获取账单 ==========`
- 查看用户信息:
  - `uid`: 用户唯一标识
  - `isLogin`: 是否已登录
  - `loginType`: 登录方式

#### 1.3 手动重新登录

1. 点击右上角【退出】按钮
2. 重新输入密码登录
3. 查看是否出现 "云端不可用" 提示

### 问题 2: 手机本地保存记账后无法删除原有记账信息

**可能原因:**
1. UI 交互问题（需要点击两次确认）
2. 数据源切换问题（云端 vs 本地）

**解决方案:**

#### 2.1 确认删除操作

删除操作需要 **两次点击**:
1. 第一次点击垃圾桶图标 → 显示 "再次点击确认"
2. **3秒内**再次点击 → 确认删除

#### 2.2 检查当前数据源

在界面顶部查看数据源切换按钮:
- **云端** (蓝色) - 数据保存在 CloudBase 数据库
- **本地** (灰色) - 数据保存在浏览器 localStorage

如果在本地模式下删除，不会影响云端数据。

#### 2.3 手动滑动删除（移动端）

1. 在账目明细页面
2. 向左滑动某个账目
3. 点击红色的删除按钮
4. 确认删除

### 问题 3: 刷新后记账信息都没了，重新登录也没有

**可能原因:**
1. 本地存储被清空（浏览器清理缓存）
2. 使用了云端数据源但云端无数据
3. localStorage 被禁用或超出配额

**解决方案:**

#### 3.1 检查浏览器控制台

打开浏览器开发者工具 (F12):
1. 切换到 Console 标签
2. 查看是否有错误信息:
   - `从主存储加载账单，数量: X`
   - `从备份恢复账单，数量: X`
   - `使用示例账单`

#### 3.2 检查 localStorage

在开发者工具中:
1. 切换到 Application 标签
2. 左侧选择 Local Storage
3. 查找 `lizard-ledger-transactions` 和 `lizard-ledger-backup`
4. 如果存在，查看内容是否正确

#### 3.3 恢复备份数据

如果主存储被清空，系统会自动从备份恢复:
- 备份键名: `lizard-ledger-backup`
- 备份格式: `{ "timestamp": "ISO时间", "data": [...] }`

#### 3.4 切换数据源

如果本地数据丢失:
1. 点击【本地】按钮切换到本地存储
2. 查看是否有本地数据
3. 如果没有，说明数据确实丢失

**预防措施:**
- 定期导出 Excel 备份（【明细】→【导出 Excel】）
- 使用云端存储模式（数据永久保存）

## 🔧 通用故障排查步骤

### 步骤 1: 清除缓存并刷新

1. 关闭浏览器
2. 清除浏览器缓存
3. 重新打开应用
4. 测试功能

### 步骤 2: 查看控制台日志

1. 打开开发者工具 (F12)
2. 切换到 Console 标签
3. 查找错误信息和警告

### 步骤 3: 检查网络连接

1. 确认手机/电脑已连接网络
2. 尝试访问其他网站
3. 检查 CloudBase 服务是否正常

### 步骤 4: 重新登录

1. 退出登录
2. 关闭浏览器
3. 重新打开并登录

### 步骤 5: 切换数据源

1. 如果云端有问题，切换到本地
2. 如果本地有问题，切换到云端
3. 查看哪个数据源可以正常使用

## 📊 数据源对比

| 特性 | 本地存储 | 云端存储 |
|------|---------|---------|
| **存储位置** | 浏览器 localStorage | CloudBase 数据库 |
| **永久性** | ❌ 清空缓存会丢失 | ✅ 永久保存 |
| **跨设备** | ❌ 单设备使用 | ✅ 多设备同步 |
| **离线使用** | ✅ 完全支持 | ❌ 需要网络 |
| **备份** | 自动备份 (localStorage) | 云端自动备份 |
| **删除方式** | 清空浏览器缓存 | 删除数据库记录 |

## 💡 最佳实践

### 1. 日常使用建议

- **优先使用云端存储**: 数据更安全，支持多设备
- **定期导出备份**: 每周导出 Excel 备份
- **不要清理浏览器缓存**: 避免本地数据丢失

### 2. 手机使用建议

- **添加到主屏幕**: 快速访问应用
- **允许通知**: 接收重要提示
- **保持网络连接**: 云端同步需要网络

### 3. 故障预防

- **不要使用无痕模式**: 无痕模式不会保存数据
- **定期检查网络**: 确保云端同步正常
- **关注错误提示**: 及时处理问题

## 🆘 紧急恢复方案

### 场景 1: 本地数据完全丢失

**恢复步骤:**
1. 切换到云端数据源
2. 检查云端是否有历史数据
3. 如果有，恢复到本地
4. 如果没有，查看是否有 Excel 备份

### 场景 2: 云端数据无法访问

**恢复步骤:**
1. 切换到本地数据源
2. 查看本地是否有数据
3. 如果没有，查看 Excel 备份
4. 手动重新录入重要数据

### 场景 3: 完全无法使用

**最后手段:**
1. 卸载应用（如果是 PWA）
2. 清除浏览器所有数据
3. 重新部署应用
4. 从 Excel 备份恢复数据
